## 1. Cluster Creation Methods

There are **4 ways** to create an EKS cluster:

| Method | Description |
|--------|-------------|
| **AWS Console** | EKS → Create Cluster → Configure Info → Done |
| **CLI** ⭐ | Use `eksctl` commands (recommended) |
| **Terraform** | `.tf` configuration files |
| **YAML** | `.yml` manifest files |

## 2. Tool Installation

### 🔧 Install kubectl

```bash
curl -o kubectl https://amazon-eks.s3.us-west-2.amazonaws.com/1.19.6/2021-01-05/bin/linux/amd64/kubectl
chmod +x ./kubectl
sudo mv ./kubectl /usr/local/bin
kubectl version --short --client
```

### ☁️ Install AWS CLI (Latest)

```bash
sudo apt install unzip
curl "https://awscli.amazonaws.com/awscli-exe-linux-x86_64.zip" -o "awscliv2.zip"
unzip awscliv2.zip
sudo ./aws/install
aws --version
```

### ⚙️ Install eksctl

```bash
curl --silent --location \
  "https://github.com/weaveworks/eksctl/releases/latest/download/eksctl_$(uname -s)_amd64.tar.gz" \
  | tar xz -C /tmp
sudo mv /tmp/eksctl /usr/local/bin
eksctl version
```

## 3. Creating & Managing Clusters

### ✅ Create a Cluster (Default — 2 nodes)

```bash
eksctl create cluster \
  --name kartik-cluster \
  --region ap-south-1 \
  --node-type t2.medium \
  --zones ap-south-1a,ap-south-1b
```

> **Default behavior:**
> - Creates **2 worker nodes**
> - Each node has **80 GB** capacity
> - **No key-pair** attached (cannot SSH into worker nodes)

### ✅ Create a Cluster (Custom — 5 nodes)

```bash
eksctl create cluster \
  --name kartik-cluster \
  --region ap-south-1 \
  --node-type t2.medium \
  --zones ap-south-1a,ap-south-1b,ap-south-1c \
  --nodes 5
```

### ❌ Delete a Cluster

```bash
eksctl delete cluster --name kartik-cluster --region ap-south-1
```

### 📋 List Clusters

```bash
aws eks list-clusters --region ap-south-1
```

**Sample output:**
```json
{
    "clusters": [
        "kartik-cluster"
    ]
}
```

## 4. Basic kubectl Commands

### Node & Pod Info

```bash
kubectl get nodes
kubectl get pods
kubectl get pods -o wide       # With IP and node details
kubectl get po                 # Shorthand for pods
kubectl api-resources
```

### Explain Resources

```bash
kubectl explain pod.apiVersion
kubectl explain services.apiVersion
kubectl api-resources | grep -i deployment
```

### Apply / Delete Resources

```bash
# Apply a YAML file
kubectl apply -f <YAML_FILE>
kubectl create -f <YAML_FILE>

# Delete resources defined in a YAML file
kubectl delete -f <YAML_FILE>
```

### Describe a Pod

```bash
kubectl describe pod <POD_NAME>
```

## 5. Pod Types

| Type | Description |
|------|-------------|
| **Single Container Pod** | One container per pod |
| **Multi Container Pod** | Multiple containers (e.g., sidecar pattern) |

> 💡 **Key Facts:**
> - Every pod gets a **unique IP address**
> - When a pod is deleted, its IP is also released → Pods are **ephemeral**
> - **CNI plugin** is responsible for allocating pod IPs

## 6. Working Inside Pods

### Exec into a Single Container Pod

```bash
kubectl exec -it <POD_NAME> -- bash
```

### Exec into a Specific Container in a Multi-Container Pod

```bash
kubectl exec -it <POD_NAME> -c <CONTAINER_NAME> -- bash
# or
kubectl exec -it <POD_NAME> -c <CONTAINER_NAME> -- sh
```

### View Logs

```bash
kubectl logs -f <POD_NAME> -c <CONTAINER_NAME>
```

### Test Connectivity Inside Pod

```bash
wget -qO- http://localhost
```

## 7. Kubernetes Services

> By default, apps running inside a pod are only accessible **within the cluster**.
> Use **Kubernetes Services** to expose your app externally.

### Service Types

| Type | Access | Use Case |
|------|--------|----------|
| **ClusterIP** | Internal only (default) | Internal communication between pods |
| **NodePort** | External via Node IP + Port | Dev/testing, direct node access |
| **LoadBalancer** | External via cloud LB | Production, AWS auto-creates a Load Balancer |

> 🔢 **NodePort Range:** `30000 – 32767`
> If not specified, Kubernetes auto-assigns a port in this range.

> ☁️ **LoadBalancer** automatically creates an AWS Load Balancer in your region.
> Traffic flow: `External LB → NodePort → Pods`

## 8. YAML Examples

### 🟡 Single Pod

```yaml
# single-pod.yaml
apiVersion: v1
kind: Pod
metadata:
  name: single-pod
  labels:
    app: single-pod
spec:
  containers:
  - name: single-pod-container
    image: nginx:latest
    ports:
    - containerPort: 80
```

### 🔵 Multi-Container Pod 

```yaml
apiVersion: v1 
kind: Pod
metadata:
  name: nginx-sidecar
spec:
  containers:
  name: nginx-sidecar-container
    image: nginx:latest
    ports:
    - containerPort: 80
  
  - name: log-trailer-container
    image: busybox:latest
    command: ["sh", "-c" ]
    args: ["tail -f /var/log/nginx/access.log"]
    volumeMounts:
    - name: nginx-logs
      mountPath: /var/log/nginx
  volumes:
  - name: nginx-logs
    emptyDir: {}  
```

```bash
kubectl apply -f single-pod.yaml
kubectl delete -f single-pod.yaml
```

---

### 🔵 ClusterIP — Pod + Service

> ⚠️ Not accessible from the browser. Internal cluster use only.

```yaml
# clusterip.yaml
apiVersion: v1
kind: Pod
metadata:
  name: clusterip-pod
  labels:
    app: clusterip-pod
spec:
  containers:
  - name: clusterip-container
    image: nginx:latest
    ports:
    - containerPort: 80

---

apiVersion: v1
kind: Service
metadata:
  name: clusterip-service
spec:
  type: ClusterIP
  selector:
    app: clusterip-pod
  ports:
  - protocol: TCP
    port: 80
    targetPort: 80
```

```bash
kubectl apply -f clusterip.yaml
kubectl delete -f clusterip.yaml
```

---

### 🟢 NodePort — Pod + Service

> ✅ Accessible from the browser via Worker Node IP + NodePort.
> Works on **any worker node** — K8s routes traffic internally regardless of which node IP you use.

```yaml
# nodeport.yaml
apiVersion: v1
kind: Pod
metadata:
  name: nodeport-pod
  labels:
    app: nodeport-pod
spec:
  containers:
  - name: nodeport-container
    image: nginx:latest
    ports:
    - containerPort: 80

---

apiVersion: v1
kind: Service
metadata:
  name: nodeport-service
spec:
  type: NodePort
  selector:
    app: nodeport-pod
  ports:
  - protocol: TCP
    port: 80
    targetPort: 80
    nodePort: 30080
```

```bash
kubectl apply -f nodeport.yaml
kubectl delete -f nodeport.yaml
```

> Access: `http://<WORKER_NODE_IP>:30080`

---

### 🔴 LoadBalancer — Pod + Service

> ✅ Best for production. AWS auto-creates a Load Balancer.
> Get the external address with `kubectl get svc`.

```yaml
# loadbalancer.yaml
apiVersion: v1
kind: Pod
metadata:
  name: loadbalancer-pod
  labels:
    app: loadbalancer-pod
spec:
  containers:
  - name: loadbalancer-container
    image: nginx:latest
    ports:
    - containerPort: 80

---

apiVersion: v1
kind: Service
metadata:
  name: loadbalancer-service
spec:
  type: LoadBalancer
  selector:
    app: loadbalancer-pod
  ports:
  - protocol: TCP
    port: 80
    targetPort: 80
    nodePort: 30080
```

```bash
kubectl apply -f loadbalancer.yaml
kubectl delete -f loadbalancer.yaml

# Get the LoadBalancer external address
kubectl get svc
```

> Access: `http://<LOADBALANCER_EXTERNAL_DNS>`
