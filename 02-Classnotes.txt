kind: namespace
metadata:
  name: swiggi-app
---

apiVersion: v1
kind: Pod
metadata:
  name: swiggi-app-pod
  namespace: swiggi-app
  labels:
    app: swiggi-app
spec:
  containers:
    - name: swiggi-app-container
      image: kastrov/swiggi
      ports:
        - containerPort: 80 

---

apiVersion: v1
kind: Service
metadata:
  name: swiggi-app-service
  namespace: swiggi-app
spec:
  selector:
    app: swiggi-app
  type: NodePort
  ports:
    - protocol: TCP
      port: 80 #service port
      targetPort: 80 #container port
      nodePort: 30080

Kubernetes Controllers

A K8s controller will always maintain the required number of pods for your application HA(high availability) for your application

desired state	- 10 pods
actual state  - 10 pods - 5 pods + 5 pods

replicas

If a pod crashes, the controller notices the pods and recreates it - reconciliation

Types of Controllers;
Replication Controller
Replica Set
Deployment
DaemonSet
StatefulSet

Replication Controller - RC ensures that a specified number of pod replicas are running at any given time
If a pod fails/deletes, the RC will create a new pod to replace the deleted/failed pod

kind: ReplicationController

template ----> we will write pod information
------------------------------------------------------------------------------------------------------------
02-rc.yaml

apiVersion: v1
kind: ReplicationController
metadata:
  name: zomato-rc #name of the RC
spec:
  replicas: 2
  selector: #it is used to identify the pod
    app: zomato-app #it should match label of the pod
  template:   #here we will write pod information
    metadata:
      name: zomato-pod  #pod name #optional
      labels:
        app: zomato-app
    spec:
      containers:
        - name: zomato-container
          image: kastrov/zomato
          ports:
            - containerPort: 3000
---
apiVersion: v1
kind: Service
metadata:
  name: zomato-service
spec:
  type: LoadBalancer
  selector:
    app: zomato-app
  ports:
    - port: 80
      targetPort: 3000

03.rs.yml

apiVersion: apps/v1
kind: ReplicaSet
metadata:
  name: zomato-rs
spec:
  replicas: 3
  selector:
    matchLabels:
      app: zomato-app
  template:
    metadata:
      name: zomato-pod
      labels:
        app: zomato-app
    spec:
      containers:
        - name: zomato-container
          image: kastrov/zomato
          ports:
            - containerPort: 3000
---
apiVersion: v1
kind: Service
metadata:
  name: zomato-service
spec:
  type: NodePort
  selector:
    app: zomato-app
  ports:
    - port: 80
      targetPort: 3000

1)Rolling Update (default)

Blue-Green Deployment
Blue		- old version		- deployed	- users accessing blue version
Green		- latest version	- deployed	- switch the traffic to green version

In blue green deployment the traffic switches immediately
